<Section icon={Target} title="Step-by-Step Linux Setup Guide">

To get the game running correctly, you must follow a specific sequence of installations. Skipping the runtime installation is the most common reason for the "Kicked by BattlEye" error.

<Steps>
  <Step num="1" title="Enable Steam Play">
    Open Steam Settings, navigate to the Compatibility tab, and check "Enable Steam Play for all other titles." Select **Proton Experimental** as the default version to ensure the latest patches for 2026 titles are active.
  </Step>
  <Step num="2" title="Install BattlEye Runtime">
    Go to your Steam Library. In the search bar, ensure "Tools" is selected in the dropdown. Search for **Proton BattlEye Runtime**. If it is not installed, download it immediately. This is the bridge that allows the anti-cheat to function on Linux.
  </Step>
  <Step num="3" title="Game Installation">
    Install SAND: Raiders of Sophie from your library. The game requires approximately 11 GB of storage. It is highly recommended to install on an SSD to reduce Trampler loading times and asset streaming hitches.
  </Step>
  <Step num="4" title="Set Launch Options">
    Right-click the game, select Properties, and in the General tab, add launch options if you encounter stuttering. A common useful string is `gamemoderun %command%` if you have Feral GameMode installed.
  </Step>
  <Step num="5" title="Initial Boot">
    Launch the game. On the first run, the game will compile Vulkan shaders. This may take several minutes; do not force close the game during this process or you will experience heavy stuttering during your first expedition.
  </Step>
</Steps>

<Callout type="tip" title="Shader Pre-Caching">
Allow Steam to finish "Processing Vulkan Shaders" before hitting the Play button. In a high-stakes extraction shooter, a 1-second stutter during a PvP fight can cost you your entire Trampler and loot haul.
</Callout>

</Section>

<Section icon={Shield} title="Troubleshooting BattlEye and Connectivity">

The most common issues on Linux are related to the network and security handshake. If you find yourself unable to join a Voyage or Storm Dive session, check the following configurations.

| Error Message | Likely Cause | Solution |
|:---|:---|:---|
| **Failed to Initialize** | Missing Runtime | Install Proton BattlEye Runtime tool |
| **Connection Timed Out** | Firewall Block | Allow ports in `ufw` or `iptables` |
| **Kicked: Disallowed Program** | Background Tool | Close macro software or specific overlays |
| **Low FPS in Cities** | Shader Cache | Wait for background compilation to finish |

<Callout type="warning" title="Kernel Compatibility">
Some "Hardened" Linux kernels or specific security-focused distros may block the specific syscalls BattlEye needs. If issues persist, try booting into a standard LTS kernel.
</Callout>

**Common Fixes for Linux Users:**
- **Verify Integrity**: Use Steam's "Verify Integrity of Game Files" if the game crashes on the splash screen.
- **Proton Version Swap**: If Experimental fails, try **Proton 9.0** or the latest stable release.
- **Clean Prefix**: If you previously used a different Proton version, deleting the `compatdata` folder (ID 1431300) can resolve configuration conflicts.
